# @wzo/regex-diagram

它把正则解析成 AST、布局成铁路图，再渲染为自包含、可主题化的 SVG；同时提供
语法/语义校验和逐 token 的自然语言解释。框架无关，可用于任意 JS/TS 项目。

## 特性

- 支持现代语法：命名组、前后行断言、Unicode 属性、v 标志集合运算
- 输出自包含 SVG，用 CSS 变量即可主题化（含暗色），可直接导出
- 解析即校验：语法错误与语义问题（永不匹配的断言、空字符类）
- 逐 token 的自然语言解释
- 框架无关的纯函数 API，仅两个小运行时依赖

## 快速开始

```ts twoslash
import { explainRegex, parseRegex, regexToSvg } from '@wzo/regex-diagram'

// 一站式：正则 → 独立 SVG 字符串（不合法或永不匹配时为 null）
const svg = regexToSvg('(\\d{4})-(\\d{2})', 'g')

// 解析 + 校验：同时覆盖语法错误和语义问题
const { ok, ast, issues } = parseRegex('a$\\d')

// 逐 token 的自然语言解释
const steps = explainRegex('(?<year>\\d{4})')
```
