# API 参考

:::field-group

::field{name="regexToSvg" type="(source, flags?) → string | null"}
给定正则源串和可选 flags，一步返回独立、可主题化的 SVG 字符串；正则无法解析、或存在语义问题（如永不匹配的断言、空字符类）时返回 null。
::

::field{name="parseRegex" type="(source, flags?) → ParseResult"}
把源串解析成 regexpp AST 并校验。结果含 `ok`、`ast`（失败时为 null）以及同时覆盖语法错误和语义问题的 `issues` 数组。
::

::field{name="lintRegex" type="(source, flags?) → RegexIssue[]"}
只跑语义检查，返回 issues 数组——永不匹配的断言、空字符类等；正则干净时为空数组。
::

::field{name="explainRegex" type="(source, flags?) → ExplainItem[] | null"}
返回扁平的解释步骤列表，每步含 depth、语法类别、token 切片和自然语言描述；正则不合法时返回 null。
::

::field{name="formatExplain" type="(desc, messages?) → string"}
用一张文案表把单个 ExplainDesc 渲染成字符串；传部分表即可本地化，未填的键回退到内置英文（见"本地化解释"页）。
::

::field{name="buildDiagram" type="(ast) → Diagram"}
把解析好的 AST 布局成带坐标的铁路图模型，可直接渲染。
::

::field{name="renderToSvg" type="(diagram, flags?) → string"}
把图模型渲染成自包含的 SVG 字符串；传入 flags 可在 pattern 后展示它们。
::

::field{name="sourceColors" type="(source, flags?) → Array<SyntaxCategory | null> | null"}
返回每个源字符的语法类别（或 null），用于高亮原始 pattern 文本。
::

::field{name="sourceRanges" type="(source, flags?) → Array<[number, number] | null> | null"}
返回每个源字符对应的、覆盖它的最小铁路图节点的起止偏移——用于把光标与悬停和铁路图联动。
::

::field{name="toRegexLiteral" type="(source, flags?) → string"}
把 source 和 flags 拼成合法的正则字面量，转义斜杠并处理空 pattern。
::

:::

## 类型

```ts
// parseRegex().issues 与 lintRegex() 的返回项
interface RegexIssue {
  rule?: 'assertionNeverMatches' | 'emptyCharClass' // 稳定 id；语法错误时无此字段
  message: string // 可读文本（语法错误时为解析器原文）
  start?: number // 出错 token 在 pattern 中的偏移（已知时）
  end?: number
}

// explainRegex() 的每一行
interface ExplainItem {
  depth: number // 嵌套层级，用于缩进
  cat: SyntaxCategory // token 的语法类别（决定共享配色）
  token: string // 这一行描述的源码切片
  desc: ExplainDesc // 语言无关的描述——本地化就渲染它
  text: string // desc 的英文渲染结果，开箱即用
  start?: number
  end?: number
}

// 配色类别键——也是 --rr-syntax-* 变量的后缀
type SyntaxCategory =
  | 'literal' | 'charset' | 'class' | 'anchor' | 'quantifier'
  | 'group' | 'lookaround' | 'backref' | 'alternation'
```
